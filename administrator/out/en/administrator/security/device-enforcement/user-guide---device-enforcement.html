User Guide - Device Enforcement

Overview

CloudConnexa Device Enforcement is a Wide-area Private Cloud (WPC) security policy that strengthens your Network and Host security posture and reduces your attack surface by locking the .ovpn profile to specific devices – allowing only authorized devices to connect to your WPC.

Device Enforcement gives Owners and Administrators the power to leverage one of the key principles of ZTNA – Least Privilege – granting access to only the resources a user should have, from only the devices they are authorized to use. When combined with CloudConnexa’s inherent WPC micro-segmentation and access controls (see Adding a Network, Adding a Host, User Groups, Access Groups), Device Enforcement reduces the threat of lateral movement throughout Networks and Hosts.

Note

Device Enforcement is Off by default and has 2 active modes: Learn and Enforce and Enforce.

This functionality which is needed for enforcement to work is supported by the OpenVPN client with the following minimum versions:

Device OS

Minimum Client Version

Windows

OpenVPN Connect 3.3.7

Mac OS

OpenVPN Connect 3.4.2

Linux

OpenVPN3 v19_beta

Android

OpenVPN Connect 3.3.2

iOS

OpenVPN Connect 3.3.3

Learn and Enforce

This mode allows Owners and Administrators the option to create a Device with or without Device IDs. If a Device is created without the ID (UUID), the Device ID will propagate in the administration portal on the next successful connection, and the User will only be able to connect from that Device on future connections.

The Learn and Enforce mode can be enabled for both Automatic or Manual Profile distribution.

If Automatic Profile distribution is enabled, the Device ID is communicated to CloudConnexa on the first successful connection.

  • For new users, the newly generated Profile is associated with the UUID after the first successful connection

  • For existing users, after Learn and Enforce is turned ON, the Profile in use by the Device is associated with the Device UUID .

If Manual Profile distribution is enabled, the Device ID (UUID) is provided by the Owner / Administrator when the Device has been provisioned with the Connect Client.

Enforce

When the Enforce mode is enabled, the Owner or Administrator must associate a user’s Profile with a Device when it is created. Only users with trusted Devices can use the Profile and connect successfully.

The Enforce mode is available for both Automatic or Manual Profile distribution. When enabled, the user can import a Profile in the Connect App, only if the Device is trusted.

If Automatic Profile distribution is enabled, the Device ID (UUID) is communicated to CloudConnexa during connection.

  • For new users, the newly generated Profile is associated with the provided UUID after the first successful connection

  • For existing users, after the setting is turned ON, the Profile in use by the Device is associated with the Device UUID

    Note

    Uses are allowed to create Devices only when profile distribution is set to Automatic.

Once enabled, Owners / Administrator must associate a user’s Profile with a Device ID. Only Users with registered devices that have been configured with the correct UUID can connect successfully. Users can obtain the Device ID from their client Settings.

Warning

If the configured UUID for the Device does not match during the profile import, the profile cannot be imported. For Devices that already have a .ovpn profile, if the Device ID (UUID) provided during connection does not match the the configured device, connection will be denied.

The Device ID can be added in the Users > Devices menu by clicking the edit icon (pencil), of the Device.

Note

UUID visibility for devices that use the min required client version, the UUID will be shown, irrespective for device enforcement settings.

If Manual Profile distribution is enabled, the Device ID (UUID) is provided by the Owner / Administrator when the Device has been provisioned with the Connect Client.

Enable Device Enforcement

To enable Enforce, follow the procedure below:

  1. Access Settings > Users and click Edit.

  2. Select Learn and Enforce or Enforce.

  3. Toggle Profile Distribution to Automatic or Manual.

    1. If Automatic is selected, User will need to import the .ovpn Profile using their credentials

    2. If Manual is selected, Owners or Administrators will generate the .ovpn Profiles on the Users settings page and send it to the User.

  4. Click Update.

  5. Click Confirm.
